## Термінологічний апарат

Нижче наведено ключові поняття, необхідні для розуміння механізмів управління ресурсами ОС:

| Поняття | Технічна сутність | Функціональне призначення |
|:---|:---|:---|
| **Process** | Екземпляр програми | Об'єкт, що володіє власним адресним простором та ресурсами CPU. |
| **PID** | Дескриптор процесу | Унікальний номер, що дозволяє ядру ідентифікувати конкретну задачу. |
| **Foreground** | Активний режим | Стан, при якому процес взаємодіє з користувачем через стандартний ввід. |
| **Background** | Фонове виконання | Робота програми паралельно з іншими процесами без блокування термінала. |
| **Load Average** | Середнє навантаження | Показник черги процесів до обчислювальних потужностей системи. |

## Теоретичні основи моніторингу

### 2.1 Аналітичний інструментарій
Для аудиту системної активності Linux пропонує спектр утиліт: статичний знімок через `ps`, інтерактивний дашборд `top`, візуалізований монітор `htop` та швидку довідку про аптайм `uptime`. Глибоке налаштування цих команд доступне через вбудовані мануали (`man ps`).

### 2.2 Модальності сортування в реальному часі
Під час роботи з утилітою `top` доступне динамічне впорядкування списку за допомогою гарячих клавіш:
- **P** — виявлення найбільш ресурсомістких обчислювальних задач (%CPU).
- **M** — фільтрація процесів за ступенем утилізації оперативної пам'яті (%MEM).
- **N** — сортування за хронологією створення (номер PID).
- **T** — аналіз за сумарним часом використання процесора.

## Робота з ядром через інтерфейс /proc

### Концепція віртуальної файлової системи
Директорія `/proc` не існує на фізичному носії; це програмний інтерфейс ядра. Вона надає доступ до параметрів апаратного забезпечення та структур даних ядра в режимі реального часу. Кожна піддиректорія з числовим іменем відповідає активному PID.

### Сеансовий моніторинг користувачів
Для верифікації активних підключень ми використовували:
* `who` — базовий перелік користувачів у системі.
* `w` — деталізований звіт, що включає час активності та назви поточних команд кожного юзера.

### Системні переривання та сигнали
Управління процесами з клавіатури базується на надсиланні стандартних сигналів POSIX:
- **Ctrl + C (SIGINT)**: Негайне переривання виконання програми.
- **Ctrl + Z (SIGSTOP)**: Тимчасова зупинка задачі з переведенням її в режим очікування.
- **Ctrl + D (EOF)**: Сигнал завершення вводу даних або виходу з поточної оболонки (Shell).

## Практичний протокол керування задачами

### Алгоритм маніпуляції фоновими процесами
Ми дослідили життєвий цикл задачі за допомогою утиліт `jobs`, `fg` та `bg`.

1. **Ініціалізація:** Запущено тривалий процес `sleep 300`, що імітує фонову активність.
2. **Захоплення контролю:** Використано комбінацію `Ctrl + Z` для призупинення. Система зафіксувала стан `Stopped`.
3. **Фонове пробудження:** За допомогою команди `bg %1` ми відновили роботу процесу, звільнивши при цьому термінал для введення нових команд.
4. **Повернення в пріоритет:** Команда `fg %1` вивела процес із фонового режиму назад на передній план.
5. **Ліквідація:** Завершення задачі виконано через команду `kill`, яка надіслала процесу сигнал SIGTERM.

## Розширені відповіді на контрольні питання

### 1. Призначення та структура директорії `/proc`
Це динамічне вікно в серце операційної системи. Кожен файл тут є точкою доступу до параметрів ядра:
- `/proc/cpuinfo` — детальна специфікація процесора.
- `/proc/meminfo` — стан пам'яті, включаючи кеш та своп.
- `/proc/uptime` — точний час роботи системи з моменту останнього завантаження.

### 2. Діагностика витоків пам'яті
Найбільш ефективним методом є запуск `top` з переходом у режим сортування за пам'яттю (`Shift + M`). Також професійним методом є використання конвеєра:
`ps aux --sort=-%mem | head -n 5`, що виводить п'ять найбільш "важких" для RAM процесів.

### 3. Генеалогія та ієрархія процесів
Будь-який процес (окрім `systemd`) має свого "батька" (PPID). Ця деревоподібна структура забезпечує спадковість прав та ресурсів. Переглянути цю ієрархію можна командою `pstree`, яка наочно демонструє, як термінал bash породжує дочірні команди.

### 4. Порівняльний аналіз `top` та `ps`
`ps` — це "фотографія": вона дає точні дані на конкретну мілісекунду. `top` — це "відео": воно дозволяє спостерігати за динамікою, що корисно для виявлення короткочасних піків навантаження.

## Специфіка мобільної інфраструктури (Android)

Android використовує модифіковане ядро Linux, тому принципи моніторингу залишаються схожими, але з урахуванням енергоефективності.
- **Activity Manager**: Компонент, що контролює життєвий цикл застосунків.
- **Low Memory Killer**: Специфічний механізм Android, який примусово завершує фонові процеси при нестачі RAM.
- **Termux**: Дозволяє використовувати стандартні бінарні файли Linux (htop, top) безпосередньо на смартфоні.
